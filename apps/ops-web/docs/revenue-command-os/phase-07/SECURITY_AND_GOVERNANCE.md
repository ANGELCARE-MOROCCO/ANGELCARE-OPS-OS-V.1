# Security and governance

MZ07 remains locked to Shadow. Email, WhatsApp, calling, discount application, price commitment, capacity commitment, contract signature and payment authority remain prohibited. Every one of the 700 new commands explicitly denies all protected external and financial tools.

New tables are RLS-enabled and client access is revoked. The active-library view uses `security_invoker`, so underlying RLS remains effective. Tenant, permission, evidence freshness, fact-versus-hypothesis, tool boundary, approval boundary, output schema and audit trace validators are mandatory.
