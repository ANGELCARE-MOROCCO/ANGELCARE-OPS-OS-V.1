# Security and Governance

The Revenue Digital Twin is commercial infrastructure, not a public catalogue.

- Sensitive prices, costs and margins require Revenue OS authority.
- Browser clients cannot read/write the underlying tables directly.
- The model uses server-side controlled repositories.
- Every mutation is allow-listed and audited.
- Retirement is preferred over destructive deletion.
- Validation gaps prevent future AI systems from treating incomplete data as fact.
- External communications, discounts, proposals and contractual commitments remain outside Mega ZIP 2.
- The model stores no OpenAI key, service-role key or provider secret.
