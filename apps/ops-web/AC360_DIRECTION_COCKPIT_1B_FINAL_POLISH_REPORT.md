## What this hardening patch improves

### 1. Production wording hardening

- Removes weak visible “demo” feeling from the Direction cockpit by sanitizing school display names that contain non-production labels.
- Keeps the page customer-facing and business-oriented.
- Replaces technical/diagnostic copy with polished customer language.

### 2. Empty state quality

The cockpit no longer shows weak executive KPIs such as `—`, `0 / 100`, or raw zero-state notes when the underlying source is not populated yet.

It now uses premium empty states such as:

- `À compléter`
- `Aucune présence pointée aujourd’hui`
- `À consolider`
- `Aucun signal parent exploitable`
- `Aucune donnée financière consolidée`

### 3. Layout containment and horizontal overflow hardening

- Adds width and min-width containment on the cockpit body.
- Uses responsive `auto-fit` KPI columns instead of forcing five rigid cards.
- Adds `minWidth: 0` to critical layout containers.
- Tightens grid density to reduce clipping inside the existing AngelCare 360 shell.

### 4. More polished executive cockpit rhythm

- Refines hero wording and hierarchy.
- Tightens KPI typography and card rhythm.
- Improves command bar labels.
- Improves finance card readability.
- Improves presence card behavior when no attendance is registered.
- Replaces generic CTA labels like `Ouvrir la page` with route-specific actions.

### 5. Backend-first behavior preserved

- All values still come from the existing server data loader.
- No static business data was added.
- No fake records were added.
- No seed data was added.
- No mock arrays were added.
- No sidebar or shell logic changed.
