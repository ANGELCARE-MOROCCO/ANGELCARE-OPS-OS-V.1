`Commercial handoff → operational acceptance → onboarding → activation readiness → launch → first service → hypercare → performance → issue resolution → growth → renewal → long-term partnership.`

Mega ZIP 5 implements 40 command contracts across handoff, partner identity, onboarding, activation, first service, hypercare, performance, health, issues, corrective actions, reviews, QBR, growth, expansion, renewals, churn rescue and tenders. ## Data doctrine

- No second disconnected partner CRM.
- Partner identity is linked to the existing B2B prospect and source opportunity.
- Handoff versions preserve the closing source snapshot.
- High-impact unresolved promises block acceptance.
- Missing metrics remain visibly missing and are never fabricated.
- Unhealthy or payment-risk partners are not automatically pushed into expansion.
- Launch and tender submission require authorized approval.
- Chrome contains no Supabase service-role key and performs no direct database mutation.
